<summary>
- The `agent.benjamin-borbe.de/v1` Config CRD gains a required `spec.taskType` string field that each agent Config declares to name the single task type it handles
- `ConfigSpec.Validate` rejects a config when `taskType` is empty, contains characters outside `[a-z0-9-]`, or exceeds 63 characters — matching the DNS-label-ish naming used elsewhere in the project

- `agent/claude/k8s/agent-claude.yaml` gains `taskType: claude`, making the existing in-repo Config manifest consistent with the new required field
- `docs/agent-crd-specification.md` documents the new required field in the Fields table and in the top-level YAML example
- No routing code reads the field yet — this spec is purely additive on the schema, codegen, manifest, and docs; the hardcoded registry shortcut is untouched
</summary>
